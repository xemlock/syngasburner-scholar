Słownik pojęć Węzeł (ang. node), również użytwany jako Segment – rodzaj treści przechowywanej przez Drupala.

Instalacja

Konfiguracja

Administer → Site building → Themes → Configure → Global settings

W sekcji Display post information on należy odznaczyć typ Scholar i zapisać zmiany. Spowoduje to niewyświetlanie informacji o dacie utworzenia i autorze strony. W przypadku stron generowanych przez moduł Scholar informacje te ... bo autor nie jest wpisywany (treść strony jest generowana automatycznie na podstawie danych z bazy).

Administer → Site configuration → Scholar settings

Ustawienia modułu, można ustawić w nich szerokość obrazów wyświetlanych w preambule generowanych stron (czytaj więcej w rozdziale x), oraz formaty dat: dla daty pojednyczej, oraz zakresów dat w obrębie jednego miesiąca i w obrębie jednego roku ale w różnych miesiącach.

Filtry

Domyślnie wszystkie treści węzłów w Drupalu są filtrowane. Zestaw filtrów definiuje format danych (ang. Input format). Domyślnie dostępne są dwa formaty: Filtered HTML (1) oraz Full HTML (2). Pierwszy nie nadaje się do użycia w węzłach generowanych przez moduł Scholar z powodu odfiltrowywania większości struktury dokumentu (filtr Filter HTML usuwa większość elementów blokowych za wyjątkiem list UL, OL i DL). Drugi można użyc z pewnymi zastrzeżeniami. W skład filtrów tego formatu wchodzą dwa, które mogą powodować, że wynikowy dokument nie będzie wyglądał tak jak powinien (np. będzie brakowało w nim przejść do nowego wiersza, lub przeciwnie, będzie ich za dużo). Są to:

  • HTML Corrector - w zamierzeniu autorów miał poprawiać błędy składni HTML, w praktyce dla poprawnych danych może wygenerować kod z jeszcze większą liczbą błędów (patrz zgłoszenie)
  • Line break converter - zastępuje znaki końca linii, a także ciągi tagów BR tagami BR i P.

Ze względu na powyższe, format danych węzłów generowanych przez moduł powinien mieć wyłączone wspomniane filtry. Wyłączenie tych filtrów nie zagraża bezpieczeństwu (layout strony, złośliwy kod JavaScript), o ile jest używany wyłącznie dla węzłów modułu Scholar.

Aby dodać nowy format danych dla węzłów modułu należy przejść do:

Administer → Site configuration → Input formats → Add input format

a następnie:

  1. wpisać nazwę (np. Scholar)
  2. ze względów bezpieczeństwa nie zezwalać żadnej roli na używanie tego formatu (nie będzie go można uwybrać podczas edycji węzłów, chyba że użytkownik ma uprawnienie administer formats)
  3. zaznaczyć co najwyżej filtr URL filter, który zamienia adresy stron oraz adresy e-mail na hiperłącza.
  4. zapisać nowy format wciskając przycisk Save configuration

Następnie w ustawieniach modułu Scholar w sekcji Nodes należy wybrać nowoytworzony format danych i zapisać zmiany.

Taksonomia

Moduł Taxonomy służy do kategoryzowania treści. Aby umożliwić kategoryzowanie węzłów tworzonych w ramach modułu Scholar, konieczne jest zainstalowanie tego modułu, a następnie podpięcie jednego lub więcej istniejącego słownika do typu zawartości o nazwie Scholar.

Content management → Taxonomy

Dla wybranego z listy słownika, przechodząc do edycji należy w sekcji Content types zaznaczyć typ Scholar. Od tego momentu bedzie można przyporządkowywać tagi z tego słownika do węzłów zarządzanych przez moduł Scholar.
